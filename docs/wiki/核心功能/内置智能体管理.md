# 内置智能体管理指南

## 概述

WeKnora 内置一组开箱即用的智能体（quick-answer、smart-reasoning、deep-researcher、data-analyst、knowledge-graph-expert、document-assistant、wiki-researcher、wiki-fixer），通过 `config/builtin_agents.yaml` 声明式配置。该文件定义内置智能体的名称、描述、头像与运行时配置，启动期由 `internal/types/builtin_agent_config.go` 的 `LoadBuiltinAgentsConfig` 加载并注册到 `BuiltinAgentRegistry`。

> 内置智能体引用的模型由 [[内置模型管理]] 提供。同为内置系统配置的还有 [[内置MCP服务管理]]。

## 内置智能体列表

| ID | 模式 | 说明 | 是否在用户列表展示 |
|----|------|------|--------------------|
| `builtin-quick-answer` | quick-answer | 基于知识库的 RAG 问答 | 是 |
| `builtin-smart-reasoning` | smart-reasoning | ReAct 多步推理 + 工具调用 | 是 |
| `builtin-deep-researcher` | smart-reasoning | 深度研究 | 是 |
| `builtin-data-analyst` | smart-reasoning | CSV/Excel 数据分析与 SQL 查询 | 是 |
| `builtin-knowledge-graph-expert` | smart-reasoning | 知识图谱问答 | 是 |
| `builtin-document-assistant` | smart-reasoning | 文档助手 | 是 |
| `builtin-wiki-researcher` | smart-reasoning | Wiki 知识库问答 | 是 |
| `builtin-wiki-fixer` | smart-reasoning | Wiki 页面巡检修订 | **否**（仅程序内部调用） |

> `builtin-wiki-fixer` 被 `builtinAgentIDsOrdered`（`internal/types/custom_agent.go:559`）刻意排除，不出现在用户智能体选择列表，但仍在 `BuiltinAgentRegistry` 注册，可由 Wiki 编辑器程序化调用。

## 文件位置与加载

- 默认路径：`config/builtin_agents.yaml`（与 `builtin_models.yaml` 同目录）
- 加载入口：`types.LoadBuiltinAgentsConfig(configDir)`，启动期调用一次（`sync.Once` 保证）
- 文件不存在：no-op，沿用代码内置的硬编码默认值
- 文件解析失败：返回错误并中断加载（与内置模型的"仅 warning 跳过"不同，智能体配置错误会显式失败）
- i18n 引用解析：`ResolveBuiltinAgentPromptRefs` 在 prompt 模板加载完成后调用，把 `system_prompt_id` / `context_template_id` 解析为实际模板内容

## YAML 结构

```yaml
builtin_agents:
  - id: <string, required>                    # 如 builtin-quick-answer
    avatar: <string, optional>                # emoji 或图标名
    is_builtin: <bool>                        # 通常 true
    i18n:                                     # 多语言名称/描述
      default:                                # 兜底语言
        name: <string>
        description: <string>
      zh-CN:
        name: <string>
        description: <string>
      # 还可配 zh-TW / ja-JP / ko-KR 等
    config:                                   # CustomAgentConfig，见下文字段
      agent_mode: <quick-answer | smart-reasoning>
      agent_type: <rag-qa | wiki-qa | hybrid-rag-wiki | data-analysis | custom>
      # ... 其余配置字段
```

### i18n 解析优先级

`resolveI18n`（`builtin_agent_config.go:160`）按以下顺序匹配 locale：

1. 精确匹配（如 `zh-CN`）
2. 仅语言前缀匹配（`zh-CN` -> 先试 `zh`，再试任意 `zh` 开头的 key）
3. `default` key
4. map 中的第一个条目

## config 段可配的模型字段

`config` 段对应 `CustomAgentConfig`（`internal/types/custom_agent.go:96`）。与模型相关的字段有 4 个：

| 字段 | YAML key | 说明 |
|------|----------|------|
| 主对话模型 | `model_id` | 智能体对话用的 KnowledgeQA 模型 ID |
| 重排序模型 | `rerank_model_id` | 检索用的 Rerank 模型 ID |
| 视觉模型 | `vlm_model_id` | 图像分析的 VLM 模型 ID（可选，缺省回退空间级 VLM） |
| 语音模型 | `asr_model_id` | 语音转写的 ASR 模型 ID（可选） |
| 查询理解模型 | `query_understand_model_id` | 查询改写/意图识别专用模型，空则回退主对话模型 |
| 追问模型 | `follow_ups.model_id` | 上下文追问问题生成专用模型（`question_suggestions.follow_ups` 下） |

> **仓库默认 YAML 不填写 `model_id` 等字段**：内置智能体本身不绑定具体模型，而是由各空间在运行时为智能体选择模型。`config` 段"可以"预配这些字段作为某个部署的默认值，但通常留空交给空间级配置。

## 模型强制要求

并非所有智能体都强制要求模型。以下三处校验决定了一个智能体能否被使用 / 分享：

### 1. 主对话模型（所有智能体必需）

- **quick-answer 模式**（`session_qa_helpers.go:107`）：智能体的 `model_id` 为空时直接报错 `chat model is not configured: please set model_id on agent <id>`；且该模型必须存在且 `type == KnowledgeQA`
- **smart-reasoning 模式**（`session_agent_qa.go`）：同样要求 `model_id` 非空且可用
- **分享校验**（`agent_share.go:102`）：分享智能体到组织前，`ModelID == ""` 会拒绝并返回 `ErrAgentNotConfigured`

### 2. 重排序模型（仅特定智能体必需）

由 `agentRequiresRerankModel`（`agent_share.go:40`）判定。**当且仅当**满足以下条件时强制要求 `rerank_model_id`：

- `kb_selection_mode != "none"`（知识库范围未禁用）
- **且** `allowed_tools` 含 `knowledge_search`（工具列表为空时回退 `tools.DefaultAllowedTools()`，其中包含 `knowledge_search`）

运行期校验在 `session_agent_qa.go:96-109`：需要 rerank 但未配置时报 `rerank model is not configured: please set rerank_model_id on the agent`。

> **Wiki 类智能体不需要 rerank 模型**：`builtin-wiki-researcher` / `builtin-wiki-fixer` 的 `allowed_tools` 只含 `wiki_*` 工具（`wiki_search`、`wiki_read_page` 等），不含 `knowledge_search`，因此即便挂载了知识库也不强制要求 rerank 模型。`kb_selection_mode: "none"` 的智能体同理豁免。

### 3. VLM / ASR 模型（可选）

仅当智能体启用了图像上传（`image_upload_enabled: true`）或语音上传（`audio_upload_enabled: true`）时才需要对应模型，否则留空即可。

## quick-answer 与 smart-reasoning 的模型解析差异

两种模式对 `model_id` 的解析路径不同：

### quick-answer（RAG 模式）

走 `sessionService.resolveChatModelID`（`session_qa_helpers.go:95`），优先级：

1. **请求级 `summary_model_id` 覆盖**：若请求带了有效的 `summary_model_id`（存在且为 KnowledgeQA 类型），优先用它
2. **智能体 `model_id`**：上述为空则用智能体配置的 `model_id`
3. **系统回退**：无智能体时走 `selectChatModelID`（KB / 会话 / 系统级回退）

> 智能体存在时，`model_id` 为空或模型不可用都会**直接报错**，不会静默回退--避免让一个未配置的智能体"看起来能用"。

### smart-reasoning（ReAct 模式）

走 `session_agent_qa.go`：

1. 使用 `effectiveModelID`（已校验过智能体 `model_id` 非空且可用）
2. rerank 模型**只从智能体配置取**（`rerank_model_id`），不再回退租户级 `ConversationConfig.RerankModelID`--强制把配置放在用户实际编辑智能体的地方，便于排查检索质量
3. 不需要 rerank 时（`agentRequiresRerankModel` 为 false）跳过 rerank 模型初始化

> 设计差异的本质：quick-answer 允许请求级临时换模型，smart-reasoning 强调"智能体声明自己的模型栈（含 rerank）"，二者都不允许智能体在缺模型时静默运行。

## agentRequiresRerankModel 判定逻辑

```go
func agentRequiresRerankModel(agent *CustomAgent) bool {
    if agent.Config.KBSelectionMode == "none" {
        return false
    }
    allowed := agent.Config.AllowedTools
    if len(allowed) == 0 {
        allowed = tools.DefaultAllowedTools()   // 含 knowledge_search
    }
    for _, t := range allowed {
        if t == tools.ToolKnowledgeSearch {
            return true
        }
    }
    return false
}
```

判定要点：

- `KBSelectionMode == "none"` -> 永远不需要（KB 工具全失效）
- `AllowedTools` 为空 -> 回退默认工具集（含 `knowledge_search`）-> **需要**
- `AllowedTools` 显式含 `knowledge_search` -> **需要**
- `AllowedTools` 显式不含 `knowledge_search`（如纯 Wiki / 纯数据分析）-> **不需要**

这与运行期 `session_agent_qa.go` 的实际调用路径严格对齐：只有 `knowledge_search` 配合启用 KB 范围才会触发 reranker。

## 注意事项

1. **`agent_mode` 决定一切**：`quick-answer` 与 `smart-reasoning` 走完全不同的代码路径，模型解析、工具调用、多轮历史处理都不同
2. **smart-reasoning 默认强制多轮**：`EnsureDefaults` 会把 `multi_turn_enabled` 置为 true（`custom_agent.go:514`），YAML 里配 false 也会被覆盖
3. **`thinking` 默认显式 false**：未配时 `EnsureDefaults` 注入 `*false`，保证厂商专属的 `thinking_control` 线格式始终收到值
4. **`citation_enabled` 默认 true**：历史智能体（字段引入前创建）默认开启引用，显式 false 才关闭
5. **内置智能体不可删除**：它们由 YAML 注册，不通过 SQL 管理；要从用户列表隐藏某个内置智能体，需修改 `builtinAgentIDsOrdered`（代码层）而非 YAML
6. **prompt 引用解析**：`system_prompt_id` / `context_template_id` 指向 `config/prompt_templates/` 下的模板 ID，启动期解析；引用的模板不存在会打 Warning 但不阻断
7. **`agent_type` 预设**：`rag-qa` / `wiki-qa` / `hybrid-rag-wiki` / `data-analysis` / `custom`，仅 smart-reasoning 模式生效，空或未知值按 `custom` 处理
